# 评测迭代方法论 (Evaluation-Iteration Skill)

> 可复用的算法优化方法论。每个优化方向遵循 5 步循环。

## 循环框架

```
Step 1: 写评测要求     →  定义指标、基线、成功标准
Step 2: 建接口+数据集   →  实现评估接口, 准备 train/test 数据
Step 3: 运行评测        →  执行基准测试, 记录全部指标
Step 4: 分析差距        →  诊断失败模式, 形成假设
Step 5: 迭代优化        →  按假设修改, 回到 Step 3
```

## Step 1: 评测要求模板

```markdown
## 评测要求: [优化方向名称]

### 指标
- 主指标: [具体指标, 如 Hit@5, MRR, Accuracy]
- 辅助指标: [如 coverage, margin, calibration]
- 对比基线: [当前最好结果]

### 数据集
- 训练: [n 样本, 来源]
- 验证: [n 样本, 来源]  
- 测试: [n 样本, 来源, 严格隔离]

### 成功标准
- 突破: [主指标提升 >X%]
- 有效: [主指标提升 >Y%]
- 无效: [无统计显著差异]

### 迭代记录
| Round | 改动 | 主指标 | Δ | 洞察 |
|-------|------|--------|---|------|
| 0 | baseline | X.XX | — | 初始基线 |
| 1 | ... | ... | ... | ... |
```

## Step 3: 运行评测

```bash
python research/iteration_runner.py --task [ltr|classifier|...] --round N
```

自动:
1. 加载基线结果 (round 0)
2. 运行当前 round 的训练+评估
3. 记录到 `research/iteration_log.json`
4. 输出对比表格

## Step 4: 分析模板

```markdown
### Round N 分析

| 指标 | Baseline | Current | Δ |
|------|----------|---------|---|

**成功案例** (改进的样本):
- case_id: 为什么之前失败, 现在成功

**失败案例** (仍失败的样本):
- case_id: 失败原因诊断

**假设** (下一轮改进方向):
- [具体假设]
```

## 方法论检查清单

- [ ] 训练集和测试集是否严格隔离 (按 comment_id)?
- [ ] 每次迭代是否只改一个变量?
- [ ] 是否记录了完整的超参配置?
- [ ] 是否分析了成功和失败的具体 case?
- [ ] 下一轮假设是否有明确的验证方式?
- [ ] 基线是否可复现 (固定 seed)?
