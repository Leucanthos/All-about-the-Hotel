# 多模态召回 — 接口与测评要求

## 一、接口设计

### 基础接口

```
prompt2image(prompt, topK=k) -> {"images": [...], "score": [...]}
image2text(image, topK=k)    -> {"texts": [...], "score": [...]}
```

### v2 增强 (基于大规模测试洞察)

| 能力 | 方法 | 驱动洞察 |
|------|------|---------|
| 多粒度召回 | `prompt2image_granular(granularity="room_type")` | Strict Hit@10=2% vs Relaxed=50% |
| 多样性重排 | `prompt2image_diverse(diversity_lambda=0.5)` | 结果 CV=0.02 高度集中 |
| 查询改写 | `auto_rewrite(prompt)` | 短文本 MRR 是长文本 2.4x |
| 置信度估计 | `get_confidence(result)` | ECE=0.48 分数不可靠 |
| 环路一致性 | `check_consistency(text, image)` | I→T→I 对称率 80%→17% |

## 三、关键指标

| 层级 | 指标 | 说明 |
|------|------|------|
| L1 准确性 | Strict Hit@K, Relaxed Hit@K, MRR, NDCG | 精确 + 松弛匹配 |
| L2 质量 | Coverage, Hard Neg Margin, ECE, Diversity CV | 索引利用 + 区分度 + 校准 |
| L3 一致性 | Symmetry (I→T→I, T→I→T), Cross-Modal Rank | 跨模态对齐 |

## 四、关键问题

1. **对称性**: prompt→image 和 image→text 是否对称？ → 不对称 (I→T→I 80% vs T→I→T 12%)
2. **最佳文本形式**: 哪种文本检索效果最好？ → 房型名称 > 查询词 >> 完整评论
3. **Score 阈值**: 卡阈值后指标提升？ → 固定阈值无效 (ECE=0.48)
4. **多粒度匹配**: Strict vs Relaxed 差异？ → 39x 差距
5. **规模效应**: Hit@K 随索引规模的变化？ → 近似幂律衰减, Relaxed 相对稳定
