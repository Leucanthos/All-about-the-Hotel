# 《大模型技术原理与商业应用》期末项目指导意见

## 一、项目选题

可选择课程指定项目 **"酒店评论智能问答系统"** 或自主选题。自主选题需与课程内容相关。

## 二、项目提交

**截止时间**：以 eLearning 为准。

**需提交内容**：

| 序号 | 内容 | 数量 | 备注 |
|:---:|------|:---:|------|
| 1 | 技术报告 | 每组一份 | 需写明分工，不超过 10 页 |
| 2 | 完整代码 | 每组一份 | 已上传 GitHub 则无需提交 |
| 3 | 关键提示词 | **每人一份** | 整理为 MD/PDF，不超过 3 页 |
| 4 | GitHub 链接 | 每组一份 | *可选 |
| 5 | 演示视频 | 每组一份 | *可选，不超过 2 分钟 |

**注意事项**：

1. **关键提示词**：筛选项目过程中的关键提示词即可，无需附带模型回复。相比代码，带有你思考的提示词更适合作为过程评价参考。

2. **GitHub 与视频**：技术报告和关键提示词可单独提交，也可附在 GitHub 中。演示视频可上传至 GitHub README，不单独提交。

3. **网站部署**：有条件的同学可部署至 Vercel、Zeabur 等平台（注意 Token 消耗）。功能仅限课程项目，**不得进行商业化行为**。使用 ≥2 家酒店数据进行公开部署需提前征得老师同意。

## 三、项目指导

### 1. 开发方式

以下方式均可，项目评分的**主要依据是技术侧做了什么**，而非开发方式。

| 方式 | 说明 | 适合人群 |
|------|------|---------|
| 全栈自研 | 自主完成所有开发 | 想做完整 Web 应用的同学 |
| 基于前端模板 | 在现有前端模板上继续开发（未集成 RAG） | 希望跳过前端工作的同学 |
| 基于全栈模板 | 在全栈模板上继续开发（已集成 RAG） | 想快速起手的同学 |
| 常规课程项目 | 正常提交报告 + 代码 + 提示词 | 对全栈/GitHub 无兴趣的同学 |

> 模板仓库：[github.com/Scorpioyyy/hotel-review-rag](https://github.com/Scorpioyyy/hotel-review-rag)，可使用 `001`/`002` 分支或 `v1.0.0`/`v1.1.2` 版本。
> 体验地址：[hotel-review-rag-scorpio.vercel.app](https://hotel-review-rag-scorpio.vercel.app/)

### 2. 可选优化方向

项目评分的主要依据是在技术侧的探索深度。建议从以下方向中选择 2-3 个进行优化。

---

#### 算法方向

| # | 方向 | 说明 |
|---|------|------|
| 1 | 数据集扩展 | 扩展到整个城市/酒旅集团的所有酒店评论 |
| 2 | 评论 Chunking | 文本分块 + 前端高亮引用 |
| 3 | 小模型替代 | 质量评分/类别划分用 BERT 等小模型实现，需评估效果 |
| 4 | 聚类分类 | 用数据挖掘（聚类）替代人为指定类别，解决多标签问题 |
| 5 | doc2query 小模型 | 用专用小模型替代 LLM 做反向 Query 生成 |
| 6 | 摘要优化 | 改进摘要生成质量 + 摘要检索策略 |
| 7 | 倒排索引升级 | 迁移至 Elasticsearch |
| 8 | 向量数据库选型 | 梳理主流向量库，替换/优化 |
| 9 | 复杂 Query 理解 | 意图改写，是否需训练专用小模型 |
| 10 | 约束检测增强 | LLM 生成过滤语句 → 更精确的混合检索 |
| 11 | HyDE 优化 | 延迟 vs 性能的平衡 |
| 12 | 多模态检索 | 注意原始图片分辨率较低 |
| 13 | 层次化检索 | 建议搭配数据集扩展 |
| 14 | LTR 替代 RRF | Learning to Rank 模型训练 |
| 15 | 多样性重排 | MMR / DPP 算法 |
| 16 | 回复提示词优化 | 结构化回复模板 |
| 17 | 自动化评估 | 用 RAGAS 等工具评估（效果未必好） |
| 18 | Agentic RAG | 多策略自选 → 评估 → 不满意则调整重试，权衡延迟 |
| 19 | 多轮对话上下文 | 上下文管理策略 |

---

#### 数据方向（ToB / ToG）

**方向 A — 数据看板增强**：丰富看板页面 + 点击交互跳转。用户点击时间序列中的异常点（如某月评分骤降），后台自动调用大模型进行：① 意图识别 ② 结合评论文本自动分析。

**方向 B — 区域/集团分析**：扩展至某城市/某集团全部酒店，评论对比 + 时间序列分析，为决策者提供实时智能分析。

> **注**：若使用 LLM 生成标注数据，请在技术报告中说明数据生成过程及标注质量评估方法。
