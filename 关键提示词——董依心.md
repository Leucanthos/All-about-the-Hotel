## 1. 聚类标签生成 Prompt

**用途**：将 HDBSCAN 聚类的结果转化为业务可读的标签格式【一级分类 - 二级细分】

**来源**：`clustering_categories.ipynb` - `generate_cluster_label()`

```prompt
你是一个专业的酒店文本挖掘与业务分析师。你的任务是根据给定的聚类特征关键词和代表性评论，生成一个规范、具体的二级结构化标签。

【核心任务】
将该簇归入以下四个法定的一级分类之一，并根据该簇的实际业务场景，独立提炼出一个具体、干净的二级细分名称。

【法定一级分类标准（严格限定这四类）】
1. 设施类：指酒店的硬件、建筑、客房设备、餐饮硬件等。
2. 服务类：指员工的主动服务、办事效率、客情维系、特殊布置等行为。
3. 位置类：指周边的交通、地铁、地理位置、周边配套或窗外景观。
4. 体验类：指顾客对环境的宏观感知，如卫生状况、安静程度、节日氛围、整体满意度。

【命名规范】
1. 必须严格采用"【一级分类 - 二级细分】"的格式。
2. 二级细分禁止泛化！禁止包含"体验"、"设施"、"服务"等模糊词。必须体现具体业务。
3. 二级细分字数必须为4~5个字，不能超过5个字。
4. 正确示例：
   - 出现"地铁, 出行, 方便" -> 【位置类 - 交通便利性】
   - 出现"卫生, 干净, 异味" -> 【体验类 - 卫生状况】
   - 出现"酒廊, 行政, 下午茶" -> 【设施类 - 行政酒廊】
   - 出现"生日, 蛋糕, 惊喜" -> 【服务类 - 生日客情】
   - 出现"施工, 噪音" -> 【体验类 - 施工噪音】

【已使用的标签（你的输出必须与以下所有标签的二级细分完全不同）】
{existing_str}

【输入数据】
- 聚类判别关键词（按 TF-IDF 权重降序）：
{kw_str}
- 代表性评论样本：
{smp_str}

【重要防错指令】
1. 关键词列表中排名最靠前的词最能代表该簇的核心主题，优先据此命名。
2. 不要在标签后面附加"（簇X）"等任何数字后缀。
3. 必须输出且仅输出标准的 JSON 字符串，禁止包含任何 Markdown 格式标记。

输出格式：
{"category": "【体验类 - 施工噪音】", "keywords": "施工,噪音,外面", "summary": "一句话描述。"}
```

**参数**：
- `model`: qwen-turbo
- `temperature`: 0.2
- `max_tokens`: 450

## 2. QA 变体生成 Prompt

**用途**：为每个类别摘要生成 3 条"回答式"短句，作为检索桥接层，缩短 Query-Document Gap

**来源**：`Database.ipynb` - `_generate_qa_variants()`

```prompt
请为酒店点评类别「{category}」生成3个简短的回答式短句，用于检索。要求：
1. 每句 20-50 字，描述该类别的核心事实
2. 风格类似对用户问句的直接回答（如"该酒店游泳池水温适宜，定期消毒，整洁度较好"）
3. 三句分别覆盖：主要优点、需注意的问题、一个具体特色细节

参考摘要（前 500 字）：
{summary_text[:500]}

直接输出 3 句，每行一句，不加序号或符号前缀：
```

**参数**：
- `model`: qwen-turbo
- `temperature`: 0.4
- `max_tokens`: 300

## 3. 摘要分层生成 Prompt（参考技术报告）

**用途**：从大量评论中生成分类别的摘要

**分层聚合流程**：

1. **分批摘要（Batch Summarization）**：将每类别的评论按批次（30-50条）输入 qwen-turbo，生成批次小结
2. **关键词整合**：对批次小结做关键词频次统计，提取 top-5 代表性关键词
3. **最终摘要合成**：将所有批次小结和关键词集中，调用 LLM 生成最终摘要（500-1000字）
